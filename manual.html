1. 시작하기

Dental Ops AI는 치과 상담실장·원장·코디가 함께 쓰는 운영 자동화 플랫폼입니다. 최초 설정은 3단계입니다.

1.1 로그인

홈 우측 상단의 로그인 버튼을 눌러 이름과 역할을 선택합니다. 역할(원장/상담실장/코디/위생사/관리자)에 따라 접근 가능한 메뉴가 달라집니다.

1.2 Gemini API 키 설정

사이드바 하단 Gemini API 키 설정 버튼을 눌러 Google AI Studio에서 발급받은 키를 입력합니다. 키는 브라우저 로컬에만 저장됩니다.

2. 8대 엔진 소개

플랫폼은 8개의 독립 엔진으로 구성됩니다. 각 엔진은 사이드바 메뉴에서 바로 접근할 수 있습니다.

💬

상담 AI (ConsultEngine)

환자 질문을 구조화된 상담 응답으로 변환

환자가 던진 질문에 대해 "직접 답변 → 추천 치료 → 예상 비용/기간 → 다음 액션" 4단계 구조로 답합니다. 증상 분류(Triage)도 지원합니다.

🎯

전환 전략 (ConversionEngine)

반대/거절 상황에서 감정에 공감하며 설득

"공감 → 재해석 → 해소 정보 → 대안 → CTA" 5단계로 환자 반대를 풀어냅니다. 전환 확률도 0~100 점수로 예측합니다.

⚙

자동화 (AutomationEngine)

반복 메시지를 템플릿으로 자동 발송

신환 환영, 상담 후속, 시술 전/후 알림, 재방문 유도 등 5종 템플릿을 지원합니다. 커스텀 메시지는 Gemini가 생성합니다.

👥

환자 관리 (Patient Management)

환자 CRUD + 상태 추적

환자의 신상/연락처/견적/상태(상담대기→치료완료)를 관리합니다. Supabase 연결 시 영구 저장됩니다.

📊

KPI 대시보드 (KPIEngine)

전환율·재방문율·AI 활용률 실시간 모니터링

6개 핵심 KPI를 한 화면에 표시하고, 목표 대비 달성 여부를 벤치마크로 보여줍니다.

🎓

교육/훈련 (TrainingEngine)

시나리오 기반 상담 롤플레이 + AI 평가

5종 상담 시나리오에 대해 교육생이 응답하면 공감·정확성·설득·대안·CTA 5축으로 100점 만점 채점합니다.

🧠

인사이트 리포트 (InsightEngine)

6단계 하네스로 데이터에서 통찰 추출

Curator → Insight Miner → Architect → Writer → Critic → Editor 순으로 리포트를 생성합니다. 패턴/역설/전략/추적 KPI를 자동으로 뽑아냅니다.

👔

CEO Advisory (CEOEngine)

원장·관리자 전용 전략 조언

KPI와 컨텍스트를 바탕으로 growth/steady/caution/danger 4단계 상태 진단, Top 3 액션, 리스크, ROI 추정을 제공합니다.

3. 상담 방법론 — 5단계 × QLRCQ

본 플랫폼의 상담 코칭은 덴탈클리닉파인더(Dental Clinic Finder)의 공식 상담 방법론을 엔진·평가 기준으로 내장했습니다. 모든 코칭 답변과 세션 평가는 아래 5단계 × QLRCQ 구조를 따릅니다.

3.1 대단계 5개 (Macro Phases)

상담은 1단계에서 5단계로 나선형 심화합니다. 단계를 건너뛰지 않습니다.

# 단계 영문 핵심
1공감Empathy환자 감정을 언어로 먼저 인정
2이해Understanding상황·맥락·우려를 구조적으로 파악
3선택권 제공Providing Choice2~3개 옵션을 중립적으로 제시, 결정권은 환자에게
4가치 전달Delivering Value판매 아닌 공유 톤으로 각 선택의 가치 전달
5신뢰 구축Building Trust재방문·추가 질문의 문 열어두기

3.2 QLRCQ 마이크로 사이클

각 대단계 내부에서 Q → L → R → C → Q' 순서의 사이클을 돕니다. 이 방법론의 차별화 포인트는 마지막의 Q' (Re-Question)입니다. Confirm에서 끝내지 않고 다시 질문으로 문을 열어 대화가 나선형으로 깊어집니다.

  • Q · Question — 질문으로 문 열기
  • L · Listen — 끊지 않고 완성된 반응 받기
  • R · Reaction — 판단 없이 감정 되비추기
  • C · Confirm — "제가 이해한 게 맞나요?"로 확인
  • Q' · Re-Question — 한 단계 더 깊이 파고드는 재질문

3.3 각 단계 3요소

단계마다 상담사가 참조할 3가지 표지가 있습니다.

  • ✅ Success Signals — 이 발화·반응이 나오면 단계가 잘 작동 중
  • 🎯 Core Principles — 반드시 지킬 태도 원칙
  • ❌ Critical Avoidance — 절대 피해야 할 어휘·행동 (예: "지금 안 하시면 큰일납니다")

3.4 플랫폼에서의 연결

  • 📤 전송 시 AI가 현재 발화가 어느 대단계 × QLRCQ 좌표에 있는지 판단 → 그 위치에 맞는 코칭 스크립트 생성
  • 🧭 좌측 상단 트래커에 현재 위치·원칙 준수 여부·금지 어휘 감지 여부 시각화
  • 🛑 상담 종료 시 전체 세션을 5축(공감 완결도 · 이해 깊이 · 선택권 존중 · 가치 전달 · 신뢰 구축)으로 평가 + QLRCQ 사이클 완주율·단계 분포·금지어 위반 횟수 산출
© 2025 Dental Clinic Finder — 본 상담 방법론(5단계 × QLRCQ)과 각 단계의 Success Signals / Core Principles / Critical Avoidance 프레임워크는 덴탈클리닉파인더(심재우 대표)의 저작물입니다. Dental Ops AI 플랫폼은 해당 방법론을 엔진·평가 기준으로 공식 탑재하여 사용합니다.

4. 상담 AI 사용법

사이드바의 상담 AI 메뉴에서 실시간 상담 녹취, AI 분석, 코칭 답변을 받을 수 있습니다.

4.1 기본 사용 흐름

  1. 🌐 상담 언어 선택 — 30개 언어 중 선택 (한국어·영어·중국어·일본어·스페인어·독일어·프랑스어 등)
  2. 환자 선택 (선택 사항 — 선택 시 맞춤 코칭)
  3. 🎙 마이크 버튼으로 환자·상담실장 대화 녹음 (자동 음성 인식)
  4. 또는 수동 입력 — 텍스트로 직접 입력 가능
  5. 📤 전송 클릭 → AI 즉시 분석 + 코칭 답변 생성
  6. 🛑 상담 종료 및 저장 — Supabase에 자동 저장

4.2 다언어 지원 (30개 언어)

상담 언어 드롭다운에서 원하는 언어를 선택하면:

  • 음성 인식이 선택된 언어로 변경됨 (SpeechRecognition API)
  • AI 분석
  • 코칭 답변이 선택된 언어로 완전히 생성됨

지원 언어: 한국어, English (US), 中文 (简体·繁體), 日本語, Español, Français, Deutsch, Italiano, Português, Русский, العربية, हिन्दी, Tiếng Việt, ภาษาไทย, Bahasa Indonesia, 등 총 30개 언어

4.3 실시간 상담 녹취 & 화자 분리

  • 환자와 상담실장의 대화를 자동으로 기록
  • AI가 자동으로 화자 분리 — "환자", "상담실장" 라벨링
  • 대화 중 누가 무엇을 말했는지 명확히 파악
  • 음성 인식이 잘못 인식한 부분도 수정 가능

4.4 즉시 코칭 답변 & 상세 분석

  • ⚡ 즉시 응답 (1~2초) — 상담실장이 바로 말할 수 있는 스크립트
  • 속마음 — 환자의 숨은 의도·불안·요구 파악
  • 🧭 5단계 × QLRCQ 좌표 — 현재 상담 위치와 다음 액션 제시
  • 상세 분석 (3~5초) — 치료 옵션, 주의사항, 성공 신호 등

4.5 세션 관리 & 저장

  • 상담 세션 시작 후 언어는 변경 불가 (일관성 유지)
  • 모든 대화, 코칭, 분석 결과가 Supabase에 자동 저장
  • 같은 환자의 과거 상담 이력을 참고하여 맞춤 분석

5. 전환 전략 활용법

환자가 "가격이 부담된다", "생각해보겠다" 같은 반대를 표현할 때 사용합니다.

📷 스크린샷 자리 — 전환 전략 생성 화면
  • 반대/우려 입력: 환자가 한 말을 그대로 기입
  • 치료와 예상 비용 입력
  • AI가 공감→재해석→해소정보→대안→CTA 5단계 멘트를 생성
  • 전환 확률 예측으로 긍정/부정 신호도 함께 분석

6. 자동화 설정

자동화 메뉴에서 5종 템플릿을 선택하거나 커스텀 메시지를 생성할 수 있습니다.

  • 신환 환영: 첫 방문 후 환영 메시지
  • 상담 후속: 상담 다음날 정리 메시지
  • 시술 전 알림: 예약 24시간 전 리마인더
  • 시술 후 케어: 시술 당일 주의사항
  • 재방문 유도: N주 지난 환자에게 검진 권유

발송 이력은 로컬 스토리지와 Supabase(연결 시)에 저장됩니다.

7. KPI 해석 가이드

대시보드의 6개 KPI 의미와 목표값:

  • 전환율: 상담 대비 계약 성공률. 목표 70% 이상
  • 평균 상담시간: 상담 1건 평균 소요. 목표 15분 이하 (짧을수록 효율적)
  • 재방문율: 치료 완료 후 정기검진 복귀. 목표 70% 이상
  • AI 활용률: 상담 중 AI 엔진 사용 비율. 목표 80% 이상
  • 월 매출: 계약/치료완료 건 합계
  • 오늘 상담: 대기 + 진행 환자 수

8. 교육/훈련 활용

신규 상담실장·코디를 위한 롤플레이 훈련 도구입니다. 5종 시나리오(비용부담/공포/기간고민/재방문/소아)로 반복 훈련할 수 있습니다.

  • 시나리오 선택 → 환자의 반대 멘트 확인
  • 자신의 대응 응답 작성 → AI 평가 요청
  • 공감·정확성·설득·대안·CTA 5축 점수 + 코칭 팁 수신
  • 결과는 Supabase에 저장되어 개인별 성장 추이 추적 가능

9. 인사이트 리포트 생성

주간/월간 경영 리포트를 AI가 자동 생성합니다. 6단계 하네스 방법론으로 데이터→인사이트→전략까지 한 번에.

  • 주제 입력 (예: "이번 주 전환율 하락 원인")
  • 데이터 소스 선택 (환자/상담로그/KPI)
  • 생성된 리포트: 요약, 패턴 발견, 역설, 전략, 추적 KPI
  • PDF 내보내기로 경영진 보고서 제작

10. CEO 대시보드

원장/관리자만 접근 가능한 전용 페이지입니다. 로그인 시 역할을 원장 또는 관리자 (CEO)로 선택해야 합니다.

  • 전체 KPI 요약 + 상태 진단 (성장/안정/주의/위험)
  • 컨텍스트 입력 → AI 전략 조언 Top 3 + 리스크 + ROI
  • 조직 온톨로지 그래프 + 자동화 후보 분석
  • 팀원별 성과 비교 테이블

11. FAQ

Gemini API 키는 어디서 발급받나요?
Google AI Studio에서 무료로 발급받을 수 있습니다. "Get API Key" → "Create API Key" 순서입니다.
API 키 없이도 사용할 수 있나요?
가능합니다. 키가 없으면 모든 엔진이 데모 응답(미리 준비된 예시)을 반환합니다. UI 흐름을 먼저 익힌 뒤 키를 설정하세요.
Supabase 연결 없이 환자 데이터를 저장할 수 있나요?
브라우저 로컬 스토리지에 임시 저장됩니다. 다만 기기를 바꾸거나 브라우저 데이터를 지우면 사라지므로, 장기 운영 시에는 Supabase 연결을 권장합니다.
여러 직원이 동시에 쓸 수 있나요?
Supabase 연결 시 같은 프로젝트 URL/Key를 공유하면 모든 직원이 동일한 데이터를 봅니다. 각자 다른 이름/역할로 로그인하면 활동 추적이 가능합니다.
환자 개인정보는 안전한가요?
Supabase RLS(Row Level Security)를 활성화하면 접근 제어가 가능합니다. Gemini로 보내는 프롬프트에는 최소 필요 정보만 포함되도록 설계되어 있습니다.
모바일에서도 사용 가능한가요?
모든 페이지는 반응형으로 구현되어 있어 태블릿/모바일에서도 작동합니다. 사이드바는 햄버거 메뉴로 자동 전환됩니다.
AI 응답이 이상할 때는 어떻게 하나요?
프롬프트에 더 구체적인 정보(환자 나이, 증상, 치료 이력)를 포함하세요. Gemini는 컨텍스트가 풍부할수록 정확해집니다. 이상한 응답은 console.log로 원문을 확인할 수 있습니다.

12. 문제 해결

⚠ Gemini API 키 오류 — "API key not valid"

키가 잘못 복사되었을 가능성이 큽니다. Google AI Studio에서 새 키를 발급받아 다시 입력하세요. 키 앞뒤의 공백도 확인하세요.

⚠ Supabase 연결 실패 — "relation does not exist"

테이블이 생성되지 않았습니다. sql/schema.sql 파일을 Supabase SQL Editor에서 통째로 실행하세요. patients, consult_logs, conversions, automations, training_results, insight_reports, kpi_snapshots 7개 테이블이 생성됩니다.

⚠ 데이터가 안 보여요

1) Supabase 연결 상태 확인 (사이드바 하단 초록불)  2) 브라우저 콘솔(F12)에서 에러 확인  3) 환자 리스트가 비어있다면 + 신규 환자 등록으로 추가  4) 그래도 안 되면 브라우저 캐시 삭제 후 재접속
