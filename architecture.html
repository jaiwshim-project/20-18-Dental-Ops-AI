UI 레이어 · 8대 엔진 · 데이터 레이어 · 외부 API 네 개의 층으로 분리된 모듈식 AI 운영 플랫폼입니다.

🏛 전체 구조도

🔧 기술 스택

💻

Frontend

  • HTML5 / CSS3 / Vanilla JS
  • Web Speech API — 30개 언어 음성 인식
  • Pretendard 웹폰트
  • 순수 SVG 차트 (라이브러리 0)
  • 반응형 모바일 대응
  • LocalStorage 세션/Fallback
🗄

Backend (BaaS)

  • Supabase PostgreSQL
  • 7개 테이블 + 1 Storage 버킷
  • supabase-js v2 클라이언트
  • Row Level Security 지원
  • 실시간 구독 가능
✨

AI

  • Google Gemini 2.0 Flash
  • 30개 언어 다국어 지원 — 프롬프트 언어 자동 전환
  • 텍스트 + 이미지 멀티모달
  • JSON 모드 안정 파싱
  • safeGemini / safeGeminiJson 래퍼
  • API 키 미연결 시 데모 Fallback

🔁 데이터 흐름 시퀀스

1
상담 준비 — 언어 선택 + 환자 입력

🌐 상담 언어 선택 (30개국) → 음성 인식 언어 자동 설정
환자 정보 선택 → 🎙 음성 녹취 또는 텍스트 입력

↓
2
음성 인식 & 화자 분리

Web Speech API로 선택된 언어로 음성 자동 인식
환자와 상담실장의 발화를 구분하여 turns 배열에 저장

↓
3
엔진 호출 + 언어 지시문

ConsultEngine.coachReply({ utterances, language })로 호출
프롬프트 맨 앞에 "LANGUAGE INSTRUCTION" 자동 추가 → 모든 응답을 선택된 언어로 생성하도록 지시

↓
4
Gemini API 호출

엔진이 다언어 프롬프트를 구성해 GeminiAPI.json()으로 전송
Gemini가 선택된 언어로 JSON 응답 반환

↓
4
DB 저장

SupabaseDB.saveConsultLog() 등으로 상담/전환/자동화 이력이 PostgreSQL에 영구 저장됩니다.

↓
5
UI 업데이트

결과가 DOM에 렌더링되고, 대시보드의 KPI/차트가 다음 조회 시 자동 갱신됩니다.
